# 架构设计

[English](../architecture.md) | **中文**

本文描述 Den 的内部设计、数据流和安全模型。

```

│                      客户端                           │

                          │
```

## 分层架构

### 1. API 层（`internal/api/`）

HTTP 服务器使用 [chi](https://github.com/go-chi/chi) 路由器。请求经过中间件栈后到达处理器：

```
请求 → 请求ID → 真实IP → 日志 → 恢复 → 速率限制 → CORS → 认证 → 处理器
```

**中间件：**
- **请求ID** — 为每个请求分配唯一 ID 用于追踪
- **真实IP** — 从代理头提取客户端 IP
- **日志** — 通过 `slog` 进行结构化请求/响应日志
- **恢复** — 捕获 panic，返回 500 而不是崩溃
- **速率限制** — 每密钥令牌桶速率限制
- **CORS** — 跨域资源共享头
- **认证** — 常量时间 API 密钥验证

### 2. 引擎层（`internal/engine/`）

引擎编排沙箱生命周期并执行业务规则：

- **沙箱追踪** — `sync.Map` 线程安全并发访问
- **ID 生成** — 通过 xid 生成短、可排序的 ID
- **限制执行** — 原子计数器实现最大沙箱限制
- **自动过期** — 回收器协程每 30 秒运行，销毁过期沙箱
- **状态持久化** — 沙箱元数据保存到 BoltDB
- **启动恢复** — 重启时从存储恢复沙箱，与 Docker 状态验证

**状态机：**
```
创建中 → 运行中 → 已停止
                → 错误
```

### 3. 运行时层（`internal/runtime/`）

Runtime 接口抽象容器后端。目前仅实现了 Docker，但接口允许添加其他后端（如 Firecracker、gVisor）。

### 4. Docker 运行时（`internal/runtime/docker/`）

使用官方 Docker SDK：
- **docker.go** — 容器生命周期
- **exec.go** — 命令执行
- **fs.go** — 基于 exec 的文件 I/O（`cat` 读取，`tee` 写入）
- **snapshot.go** — 通过 `docker commit` 实现快照

**为什么使用基于 exec 的文件 I/O？** Docker 的 `CopyToContainer`/`CopyFromContainer` 在 overlay 文件系统层操作，与只读根文件系统不兼容。基于 exec 的方式通过容器的挂载命名空间工作，正确处理 tmpfs 挂载。

## 安全模型

### 容器隔离

| 控制项 | 设置 |
|--------|------|
| 权限 | 丢弃 `ALL`，仅添加 `NET_BIND_SERVICE` |
| 根文件系统 | 只读 |
| 可写挂载 | tmpfs：`/tmp`、`/home/sandbox`、`/run`、`/var/tmp` |
| 特权 | `no-new-privileges` |
| PID 限制 | 默认 256 |
| 内存限制 | 默认 512MB |
| CPU 限制 | 默认 1 核 |
| 网络 | 内部 Docker 网络 |
| 端口绑定 | 仅 `127.0.0.1` |

### API 安全

| 控制项 | 实现 |
|--------|------|
| 认证 | `X-API-Key` 头 + 常量时间比较 |
| 速率限制 | 每密钥令牌桶 |
| 请求体限制 | JSON 1MB，文件上传 100MB |
| 路径验证 | 空字节拒绝 + 遍历防护 |
| 错误处理 | 客户端收到通用消息，详情记录日志 |

## 性能特征

Apple Silicon（M 系列）基准测试：

| 操作 | 延迟 |
|------|------|
| API 健康检查 | < 1ms |
| 创建沙箱 | ~100-160ms |
| 执行命令 | ~20-30ms |
| 读取文件 | ~28-30ms |
| 写入文件 | ~56-70ms |
| 并行吞吐 | ~66 req/s |

## 持久化与恢复

1. **正常运行：** 创建时保存元数据到 BoltDB
2. **优雅关闭：** 引擎销毁所有运行中的沙箱，关闭存储
3. **崩溃恢复：** 启动时加载 BoltDB 数据，验证每个容器的 Docker 状态
4. **快照：** 作为带标签的 Docker 镜像存储，元数据在 BoltDB 中
